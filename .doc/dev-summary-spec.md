# 开发总结规范

**版本:** v1.0  
**适用范围:** 所有 Task 开发完成后的总结报告

---

## 📋 开发总结模板

每次 Task 完成后，必须向 Carrick 发送以下格式的开发总结：

```markdown
## Task
- T-XXX: Task标题

## 开发内容 (n commits)
- `abc1234` - 修改了 xxx（原因/目的）
- `def5678` - 添加了 xxx（功能）
- `ghi9012` - 重构了 xxx（优化）

## 测试结果 (x/3 通过)
1. 单元测试 - ✅/⚠️跳过/❌
2. 集成测试 - ✅（含数据验证：检查假数据/TODO标记）
3. E2E关键路径 - ✅

## 部署
- ✅ 已部署到 https://xxx
- 或：待Carrick确认后部署
```

---

## 📊 状态符号说明

| 符号 | 含义 |
|:---:|:---|
| ✅ | 通过 |
| ⚠️ | 跳过（需说明原因） |
| ❌ | 失败（需修复） |

---

## 🧪 测试分层说明

### 单元测试
- 单个函数/组件测试
- 覆盖率要求：业务逻辑 ≥ 80%

### 集成测试
- 构建测试
- 数据验证（假数据/TODO标记检查）
- 类型检查
- 安全扫描

### E2E关键路径
- Task创建时定义的验收场景
- 自动化测试覆盖

---
